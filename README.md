## Versioning

We use [SemVer](http://semver.org/) for versioning. For the versions available, see the [tags on this repository](https://github.com/yourproject/repository/tags).

### Creating a New Version

1. Update the version number in `package.json` according to semantic versioning rules.
2. Run any necessary build scripts or tests to ensure functionality.
3. Commit your changes with a message like `git commit -m 'feat: add new feature'`.
4. Tag your commit with the new version number: `git tag -a v1.0.1 -m "Minor feature addition"`.
5. Push the tag with `git push origin v1.0.1`.

## Pushing Changes

1. After making your changes and committing them, push to your fork with `git push`.
2. Go to the original project repository and create a pull request.
3. Clearly describe your changes and their impact.
